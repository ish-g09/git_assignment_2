# git_assignment_2
This is for software engineering lab

Assignment Steps:
Task 1: GitHub Account & Repository Setup
• Create a GitHub account (if not already available).
• Create a new public repository named git_assignment_2.
• Initialize the repository with a README.md file.
• Clone the repository to your local system.
Task 2: Remote Repository Workflow
• Update README.md with Name, Roll No., Course, and Semester.
• Commit the changes locally with a meaningful message.
• Push the commit to the GitHub remote repository.
Task 3: Branching and Pull Requests
• Create a branch named feature/documentation.
• Add a file github.md describing at least 8 GitHub features.
• Commit and push the branch to GitHub.
• Create a Pull Request to merge this branch into the main branch.
Task 4: Collaboration with Team Members
• Add at least one collaborator to your GitHub repository.
• The collaborator must clone the repository and create a new branch named
feature/collab-update.
• The collaborator should make meaningful changes to README.md or create a new file.
• Collaborator must push the branch and open a Pull Request.
• Repository owner must review and merge the Pull Request.
Task 5: Issues and Assignment
• Create two GitHub Issues related to tasks or improvements.
• Assign one issue to yourself and one to the collaborator.
• Close at least one issue using a commit message keyword.
Task 6: Conflict Handling via GitHub
• Both owner and collaborator edit the same line in README.md from different branches.
• Create Pull Requests from both branches.
• Resolve the merge conflict using GitHub’s conflict resolution interface.
Task 7: Tags and GitHub Releases
• Create a Git tag named v2.0 locally and push it to GitHub.
• Create a GitHub Release using the v2.0 tag with release notes.
Task 8: GitHub Insights
• Explore the Insights tab of the repository.
• Observe commit history, contributors, and collaboration activity.
